Debugging

Even the best programmers need help debugging their code. Sketch 3.0 provides improved error reporting, so now finding errors is quicker than ever.

Basic Debugging

Whenever you're running some code, you can use the log() feature to show the value of a variable, or check whether an object is defined.

For example, if you want to display the value of a variable called rectangles, you'd do:

log(rectangles)

To see the output of the log() function, you need to open Console.app:

where you'll see:

08:50:39 Sketch: 4

Which means the value for rectangles is 4, but probably already knew that.

Once you start debugging more complex plugins, you'll probably want to filter log messages in Console.app, as all the system logs are output there. To do that, click on the Filter text field at the top right, and write 'Sketch'.

If you want to create a permanent filter, click File › New System Log Query… and fill the window with these values:

Then click 'Ok', and you'll have a new section called 'Sketch' on the 'System Log Queries' at the left column.

If you are more of the command line type, you can open a Terminal.app window and run this:

$ tail -f /var/log/system.log|grep Sketch

Showing debug output

Sometimes, when debugging Plugins, you may want to check the output of a command without having to switch to another app. For those cases, you have two options:

  1. you can use one of the techniques discussed in User Input & Feedback (alert, [doc showMessage] or Cocoa dialogs), or
  2. use Sam Deane's log panel, which uses Cocoa code to create a persistent log panel you can see without leaving Sketch.
